# 打卡营（camp）业务知识

## 一、核心概念

打卡营（tiger_clockincamp）是 iPlayABC 的**周期学习活动**产品。

> 把课程内容打包成"每日任务"，学生需要在规定时间内完成每天的阅读任务并打卡。

---

## 二、数据模型

### 核心表

| 表名 | 用途 |
|------|------|
| `tiger_clockincamp` | 打卡营主表 |
| `tiger_clockincamp_syllabus` | 打卡营关联的 syllabus（课程/图书） |

### tiger_clockincamp 字段

| 字段 | 说明 |
|------|------|
| id | 主键 |
| uuid | 唯一标识 |
| name | 打卡营名称 |
| day_type | 天数 = 图书数量（一期打卡营有多少天/本书） |
| status_ | 状态：1=未开始，2=进行中，3=已结束 |
| begin_time | 开始时间（格式：yyyyMMdd） |
| end_time | 结束时间 |
| seq | 序号（同一机构内的排序） |
| del | 删除标记 |

### tiger_clockincamp_syllabus 字段

| 字段 | 说明 |
|------|------|
| camp_id | 关联的打卡营 ID |
| syllabus_id | 关联的 syllabus（课程/图书）ID |
| seq | 顺序（第几天解锁哪本书） |

## 三、业务流程

### 3.1 创建打卡营

```
机构管理员创建打卡营
    ↓
填写：打卡营名称 + 天数（day_type）
    ↓
状态 = 1（未开始）
    ↓
保存 tiger_clockincamp 记录
```

### 3.2 分配图书

```
管理员为打卡营选择 syllabus（图书/课程）
    ↓
图书数量必须 = day_type（天数）
    ↓
保存 tiger_clockincamp_syllabus（按 seq 决定解锁顺序）
    ↓
第1本书 = 第1天解锁，第2本书 = 第2天解锁，以此类推
```

**关键规则：**
- 如果 `day_type != 选择的图书数量`，不能开始打卡营
- 图书顺序很重要，决定了每天的解锁内容

### 3.3 开始打卡营

```
管理员点击"开始打卡营"
    ↓
系统检查：
  1. 没有其他进行中的打卡营（同时只能有1个进行中）
  2. day_type = 已选图书数量
    ↓
计算时间：
  - begin_time = 今天
  - end_time = 今天 + day_type 天
    ↓
状态改为 2（进行中）
```

### 3.4 打卡营进行中

```
每天自动解锁一本图书
    ↓
学生进入当天解锁的图书学习 = 打卡
    ↓
系统记录学生的学习行为
```

**结束判定：**
- `end_time + 1天` 之后，系统自动将状态改为 3（已结束）

## 四、业务规则

### 4.1 同时只能有1个进行中的打卡营

```javascript
// beginCamp 中的检查
const begining = await mysql.selectOne(`tiger_clockincamp`, { status_: 2 });
if (begining) {
  return ctx.body = { msg: `${begining.name} 正在进行中，结束后才能新开打卡营` };
}
```

**原因**：避免学生同时参加多个打卡营，造成任务冲突。

### 4.2 天数 = 图书数量

```javascript
if (row.day_type != campsylbs.length) {
  return ctx.body = { msg: `需要选够${row.day_type}本书才能开始打卡营` };
}
```

**含义**：每一本图书对应一天的打卡任务，不能多也不能少。

### 4.3 图书顺序决定解锁顺序

`seq` 字段决定了每天解锁哪本书：
- seq=1 的书 → 第1天解锁
- seq=2 的书 → 第2天解锁
- ...
