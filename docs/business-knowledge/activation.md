# 图书激活（book_activation）业务知识

## 一、核心概念

book_activation 模块解决的是**"如何把实体书变成在线课程"**的问题。

场景：出版社出版了一本英语教材，附赠在线学习权限。学生买书后，刮开激活码，扫码或输入码，在线课程就开通了。

## 二、两种激活场景

### 场景 A：学生扫码激活（被动激活）

学生买了书，在 App 里扫激活码，系统自动开通课程。

**流程**：
```
学生：打开 App → 扫激活码 / 输入激活码
    ↓
系统校验：
  - 激活码是否存在？
  - 是否已被使用？（user_id 是否已有值）
  - 是否过期？（end_date vs 当前时间）
  - 是否被作废？（del=1）
    ↓
通过 → 开通课程（active_date=今天，end_date=今天+days）
失败 → 提示具体错误（码不存在/已使用/已过期/已作废）
```

**激活码格式**：
```
{org_prefix}||{16位UUID}
例如：dige||a1b2c3d4e5f6g7h8
```

**关键表**：`{book_prefix}_book_code`

### 场景 B：机构批量激活（主动开通）

机构找到一批学生，统一开通课程。不需要学生扫码。

**流程**：
```
机构管理员：在后台选择产品（syllabus）→ 输入学生手机号 → 批量开通
    ↓
系统：
  - 如果手机号未注册，自动创建 student 账号（auto_reg=1）
  - 为每个产品生成一个激活码（UUID格式）
  - 设置有效期（days，默认365天）
    ↓
开通成功，返回生成的激活码列表（可选）
```

**限制**：
- 单次最多 300 个码
- 14天试用每个产品只能激活一次

## 三、激活码生命周期

```
生成（batchGenerate）
    ↓
未使用（user_id=null）
    ↓
已激活（user_id=学生ID，active_date=激活时间，end_date=到期时间）
    OR
已作废（del=1，通过 setCodeInactive 操作）
```

**注意**：作废已激活的码**不会追溯**——已开通的课程仍然有效，只是该码不能再给其他人用。

## 四、特殊规则

### 14天试用限制
- 每个学生对一个 syllabus 只能有**一次** 14天试用机会
- 代码逻辑：查询 `{prefix}_book_code` 表，看是否有 `days=14` 且 `user_id=该学生` 的记录

### buy_540 标签
- 有些课程标记了 `buy_540` 标签
- 如果购买的是365天套餐但有 `buy_540` 标签，实际给 487 天（540-53天的折扣）
- 这是销售策略决定的有效期差异

### codeType 分支
- `type=1`：子课程（如某个级别）
- `type=0`：父课程（如整个系列）
- 不同 type 决定激活码适用的课程范围

## 五、核心 API

| 方法 | 用途 |
|------|------|
| `getList` | 查询激活码列表（按码/手机号筛选） |
| `showQrcode` | 生成激活二维码（扫码下载 App + 带code参数） |
| `setCodeInactive` | 作废激活码 |
| `getBookNodeList` | 获取可激活的产品列表（syllabus树） |
| `batchActivation` | 批量开通（机构操作） |
| `batchGenerate` | 批量生成激活码 |

## 六、数据表

**表名**：`{book_prefix}_book_code`

| 字段 | 说明 |
|------|------|
| id | 主键 |
| type | 码类型：0=父课程，1=子课程 |
| syllabus_id | 关联的课程ID |
| book_active_code | 激活码（格式：dige\|\|UUID） |
| master_code | 主码（用于码关联） |
| user_id | 激活用户ID（null=未使用） |
| creator_id | 创建者ID（机构管理员） |
| days | 有效期天数 |
| active_date | 激活时间 |
| end_date | 到期时间 |
| del | 是否作废：0=正常，1=作废 |

## 七、与 Syllabus 的关系

激活码的本质是 **Syllabus 的访问权限**。

- Syllabus = 产品（是什么课）
- 激活码 = 访问权的凭证（怎么让特定学生访问）

一个 Syllabus 可以生成多个激活码，每个码对应一个学生的访问权限。
