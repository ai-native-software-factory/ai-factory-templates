# 学生使用旅程

> 从学生角度描述 iPlayABC 平台的使用流程
> 注意：这是从代码中逆向推断的，实际产品可能有差异

---

## 一、学生账号体系

### 学生账号从哪里来

学生账号和教师账号是**两套独立的体系**：

| 账号类型 | 表 | 登录方式 |
|---------|----|---------|
| 教师/管理员 | `user` 表 | 手机号+密码 |
| 学生 | `student` 表 | 家长手机号注册 |

学生和家长的关系：
- 一个家长手机号可以绑定多个学生（兄弟姐妹）
- 学生通过 `dfzx_user_uuid` 与 `user` 表关联

### 学生如何获得课程

```
方式1：家长购买图书 → 扫码激活码
方式2：机构批量开通 → 家长无需操作
方式3：直接购买（如果有C端购买入口）
```

## 二、学生使用旅程

### 阶段 1：打开 App

```
打开 App
    ↓
登录（家长手机号 or 微信授权）
    ↓
看到"我的课程"或"学习中心"
```

**能看到什么？**
- 已激活的课程列表（来自 `book_code` 表的激活记录）
- 进行中的打卡营
- 待完成的作业
- 直播课日程（如果有）

### 阶段 2：开始学习

```
选择一门课程
    ↓
看到 syllabus 树（章节目录）
    ↓
选择具体课件（courseware）
    ↓
学习（视频/游戏/练习/阅读）
    ↓
系统记录学习进度
```

**学习进度怎么记录？**
- `dongfang_syllabus_visits` 表：记录学生进入课件的时间、心跳
- 直播课：`live_lessons_consume_details` 记录实际上课时长

### 阶段 3：作业与测评

```
收到作业通知
    ↓
打开作业（来自 emahomework）
    ↓
完成作业（跟读练习/听力/视频等）
    ↓
提交
    ↓
AI批改（如果是 mc 类型）
    OR
教师批改（其他类型）
    ↓
查看成绩
```

### 阶段 4：直播课

```
收到直播课通知
    ↓
进入教室
    ↓
参与直播（视频互动/举手/聊天）
    ↓
下课
    ↓
系统按实际上课分钟数扣减课时
```

**课时消耗：**
- 30 秒一次心跳保活
- 离开超过一定时间判定下课
- 最小计费单位：10 分钟

### 阶段 5：打卡营

```
加入打卡营
    ↓
每日签到（打卡）
    ↓
解锁当天课程内容
    ↓
完成当天任务
    ↓
营期结束 → 获得结营证书/奖励
```

## 三、关键数据记录

| 场景 | 记录表 | 关键字段 |
|------|--------|---------|
| 激活课程 | `{prefix}_book_code` | user_id, syllabus_id, active_date, end_date |
| 学习进度 | dongfang_syllabus_visits | student_id, courseware_id, enter_time, last_heartbeat |
| 直播上课 | live_lessons_consume_details | org_id, student_id, minutes, consume_time |
| 作业 | ema_homework_student_detail | student_id, homework_id, status_, score |
| Lexile 测评 | lexile_test_report | student_id, lexile_value, report_date |
